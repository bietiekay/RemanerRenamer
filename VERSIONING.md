# Versioning

RemanerRenamer uses Semantic Versioning.

## Pre-1.0

Before `1.0.0`, minor releases may refine schema behavior and generated script
structure as the product stabilizes.

Suggested milestones:

```text
0.1.0  Initial concept and static app
0.2.0  Folder loading and preview refinements
0.3.0  Schema parser and capture improvements
0.4.0  Output rendering and conflict detection refinements
0.5.0  Script generation hardening
1.0.0  First stable release
```

## Change Rules

- Major: breaking schema behavior or generated script compatibility.
- Minor: new features, conflict modes, system variables, or UI capabilities.
- Patch: bug fixes, safety corrections, documentation fixes, and localization
  corrections.
